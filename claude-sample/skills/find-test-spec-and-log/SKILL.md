# テスト資料探索＆ログ作成スキル

結合テスト・画面テストを始める前段階として、テスト関連資料（明細書・仕様書など）を探索し、
その過程と結果をログとして記録するためのスキル。CLAUDE.md の「基本方針」「結合テストの実施手順」に従う。

## 実施手順

### 1. CLAUDE.md に記載された保管場所を確認する

- まず CLAUDE.md に記載のテスト明細書保管場所（例: `D:\ITテスト`）にアクセスし、中身を確認する。
- 併せて以下の優先ディレクトリもプロジェクト内で確認する。
  - `docs/`, `document/`, `documents/`, `spec/`, `specs/`, `test/`, `tests/`

### 2. 優先キーワード・拡張子でファイルを探索する

以下の名前を含むファイルを優先的に探す。

- 結合テスト、結合試験、テスト明細書、試験項目、仕様書、要件定義、基本設計、詳細設計、integration、test、spec

以下の拡張子のファイルも確認する。

- `.xlsx`, `.xls`, `.csv`, `.docx`, `.pdf`, `.md`

### 3. 発見した資料を一覧化する

ファイル名・パス・サイズ・更新日時などをまとめる。
複数見つかった場合は一覧表として整理する。

### 4. ログを作成する

プロジェクト直下の `logs/` フォルダに、以下の命名規則でMarkdownファイルを作成する。

```
logs/<YYYY-MM-DD>_<作業内容の要約>.md
```

ログには以下の項目を含める（[2026-06-07_結合テスト開始_明細書探索ログ.md](../../../logs/2026-06-07_結合テスト開始_明細書探索ログ.md) を雛形として参照）。

- 日時
- 指示内容（ユーザーから受けた指示の要約）
- 実施した作業（探索したパス・実行したコマンド・確認した結果を時系列で）
- 発見した資料（一覧表形式：ファイル名・パス・サイズなど）
- 現状のステータス
- 次のアクション（ユーザー指示待ちの内容）

### 5. 資料を発見した時点で一旦停止し、ユーザーに指示を仰ぐ

- CLAUDE.md の方針「期待値を推測してはいけない」に従い、資料を開いて中身を確認する前に必ず停止する。
- ユーザーへ「この資料を開いてよいか」「他に確認すべき資料はないか」を確認してから次工程に進む。

### 6. 資料が見つからない場合

CLAUDE.md の指示通り、テストを開始する前にユーザーへ報告する。

> 結合テスト明細書や仕様書が見つかりませんでした。資料の場所を教えてください。

資料が見つからない状態で期待値を推測してテストを進めてはならない。
