# テスト明細書の内容把握＆ログ追記スキル

[[find-test-spec-and-log]] で発見したテスト明細書・仕様書を開いて中身を読み取り、
ヘッダー情報とテスト項目一覧を整理してログに追記するスキル。CLAUDE.md の
「ドキュメントを読んで仕様を理解する」「期待値はソースコードではなく仕様書を優先して判断する」
「期待値を推測してはいけない」という方針に従う。

## 実施手順

### 1. ファイル形式に応じた読み込み方法を選ぶ

- `.md`, `.csv`, `.txt` などのテキスト系 → Read ツールでそのまま読む
- `.xlsx`, `.xls` → Read ツールはバイナリを直接読めないため、Python + openpyxl を使う
  ```python
  import openpyxl
  wb = openpyxl.load_workbook(r"<パス>", data_only=True)
  ```
  - 出力にWindowsのコンソール文字コード問題で日本語が文字化けすることがあるため、
    結果は一旦 `logs/_xlsx_dump.txt` のようなUTF-8テキストに書き出してから Read ツールで読む。
  - `ws.merged_cells.ranges` も確認し、結合セルでヘッダーやテスト項目が分断されていないか把握する。
  - 確認が終わったら一時ダンプファイル（`_xlsx_dump*.txt` 等）は削除する。
- `.docx`, `.pdf` → 同様に対応するPythonライブラリ（python-docx, pypdf等）やツールで内容を取り出す

### 2. 内容を整理して把握する

以下を抽出・整理する。

- ヘッダー情報（システム名、サブシステム名、作成者、作成日、実施者、会社名 等）
- テスト項目一覧（No、分類、テスト項目、検証内容、実施日、結果、備考 等の列構成に沿って表形式に整理）
- 実施済み／未実施の状態（「結果」列が埋まっているか）

**注意：ここで期待値を推測したり補完したりしてはならない。仕様書に書かれている内容のみを転記する。**
記載が曖昧・不足している場合は、その旨をそのままログと報告に残す。

### 3. ログに追記する

[[find-test-spec-and-log]] で作成した当日のログファイル（`logs/<YYYY-MM-DD>_*.md`）に
「追記：仕様書の中身確認」のようなセクションを追加する。含める内容：

- 確認方法（使用したツール・ライブラリ）
- ヘッダー情報の表
- テスト項目一覧の表
- 気づいた点（未実施項目、空欄、記載の曖昧さなど）
- 更新後のステータス（仕様の理解状況、次に何をすべきか）

### 4. ユーザーに報告し、次工程の承認を得てから停止する

把握した内容（ヘッダー情報、テスト項目一覧）を簡潔にまとめてユーザーに提示し、
「この内容でテストを開始してよいか」を確認してから一旦停止する。
ユーザーの承認なしにテスト実施（Playwright操作等）に進んではならない。
